API

Application Programming Interface

Conjunto de regras e definições que permite que diferentes softwares se comuniquem entre si.

Tipos de API

  • RESTful
    • Estilo de arquitetura para sistemas distribuídos, que utiliza os verbos HTTP para comunicação entre Client e Server.
    • GET - Recuperar dados. O servidor envia somente a informação solicitada.
    • POST - Criar novos recursos.
    • PUT / PATCH
      • PUT → Atualização completa
      • PATCH → Atualização parcial
    • DELETE - Remoção de dados do Servidor
  • SOAP
    • Independente de linguagem e plataforma de transporte
    • Utiliza XML - Mais estruturado e verboso em comparação com o JSON
    • Requisitos mais rigorosos - Exige um cabeçalho para definir aspectos como autenticação e segurança
  • GraphQL
    • Consulta personalizada - Ao contrário do REST, o cliente pode especificar exatamente quais dados ele deseja, reduzindo o tráfego de dados
    • Eficiente na redução de solicitações e no tamanho dos dados transferidos
    • Flexível e fortemente tipada, facilitando a evolução das APIs
